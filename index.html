Data assets

What the database contains

The homepage highlights the core assets. Full source tables, activity metrics, and record previews are kept in the analysis view.

Formulation

Formulation and component records

Ionizable/helper/sterol/PEG components, ratio fields, and source metadata for lookup, deduplication, and formulation-space mapping.

Structure

Ionizable-lipid structures

Unified SMILES, molecular descriptors, and ChemBERTa embeddings for structural similarity, structure-activity modeling, and candidate ranking.

mRNA delivery

mRNA delivery subset

mRNA-related records separated from the full database while retaining payload, cell/tissue, route, and readout context.

Quality

Physicochemical and particle quality

Size, PDI, zeta, encapsulation, and related readouts managed separately for quality, stability, and manufacturability modeling.

Workflow

From public data to modeling-ready tasks

This page explains how the data is organized without placing every audit table on the landing view.

1

Integrate public sources

Collect LNP records from supplementary files, GitHub, Zenodo, and published databases.

2

Normalize entities

Standardize source, SMILES, components, payload, readout, and experimental context fields.

3

Build task subsets

Split records by structure-activity, mRNA delivery, physicochemical quality, and formulation-space use.

4

Run baseline analysis

Use RDKit descriptors and ChemBERTa embeddings to check modeling readiness and extrapolation risk.

Modeling use cases

What these data can support

Structure → Activity

Structure-activity modeling

Tests whether ionizable-lipid structure carries delivery or expression signal. Current use is baseline modeling, extrapolation checks, and coarse candidate ranking.

Payload-specific subset

mRNA delivery subset

Separates mRNA records while retaining cell, tissue, route, and readout context so unlike experimental labels are not pooled into one target.

Formulation space

Formulation and quality readouts

Moves from single lipid structures to full formulation records for similarity search, gap analysis, and particle-quality modeling.

Current baseline

The deep learning layer is a modeling-readiness check

The current model is not a final SOTA delivery predictor. It is a frozen ChemBERTa plus RDKit-descriptor baseline for inspecting molecular representation, task splits, source differences, and extrapolation risk.
